Project Title: House/Mess Finder & Comparison Tracker
Tech Stack: Next.js (App Router), Tailwind CSS, Supabase (Database & Storage), Lucide React (Icons).

Core Objective
Create a single-page web application to help me track and compare houses, sublets, or mess seats I find on Facebook/WhatsApp. The app must store data in Supabase and allow image uploads for each listing.

1. Database Schema (Supabase)
Create a table named listings with the following fields:

id: UUID (Primary Key)

location: Text (e.g., Mohakhali Wireless)

category: Text (Sublet/Mess/Flat/Seat)

rent: Numeric

is_negotiable: Boolean

bills: JSONB (Electricity, WiFi, Maid, Waste)

bathroom: Text (Attached/Common)

roommates: Integer

distance: Text (Distance from University)

pros: Text[] (Array of strings)

cons: Text[] (Array of strings)

images: Text[] (Array of image URLs stored in Supabase Bucket)

contact_info: JSONB (Name and Mobile Number)

google_map_url: Text (Optional)

2. UI/UX Requirements (v0 & Frontend)
Modern Dashboard: A clean, mobile-responsive grid layout.

Image-First Cards: Each card must show the primary image, price (Rent + Total Bills), and location.

Add New Listing Form: A multi-step or modal form to input all details, including a multi-image uploader that pushes files to Supabase Storage.

Sorting & Filtering: Ability to sort by Rent (Low to High) and Distance.

Comparison Engine:

Add a "Select to Compare" checkbox on each card.

A floating "Compare" button that opens a side-by-side table comparing features of selected items.

3. Key Features to Implement
Share Button: A button on each listing to copy a formatted summary (text) to the clipboard for sharing on WhatsApp/Messenger.

Image Gallery: A light-box or carousel to view all uploaded photos for a specific listing.

Map Link: If a URL is provided, show a "View on Maps" button.

4. Design Language
Clean, minimalist aesthetic (like Airbnb or Notion).

Use Shadcn/ui components for buttons, dialogs, and inputs.

Primary color: Blue or Teal.
